# Prose Discipline Hook

코드가 아닌 파일(Markdown)의 산문 위생을 담당하는 `Edit|Write|MultiEdit` 대상 `PostToolUse` hook.
두 가지를 강제한다: hard break는 의미 경계에 오고, 산문을 상당량 추가하면 간결함 재검사를 받는다.

## 왜 존재하나

prose-style 규칙이 이미 두 관심사를 다루지만, 상주 규칙은 맥락 전체와 경쟁하다 진다 — `comment-discipline`을 hook으로 만든 것과 같은 이유다.
두 실패 모드가 반복된다.
이미 특정 컬럼으로 hard-wrap 된 파일을 편집할 때 기존 스타일을 따라 하는 것이 좋은 매너처럼 읽혀서, 새 텍스트가 한계에서 감긴다 — 관사가 명사와 떨어지고 괄호 묶음이 쪼개진다.
그리고 산문은 쌓인다: 추가되는 문장 하나하나는 따로 보면 정당해 보여서, 문서는 어느 한 편집도 책임지지 않는 장황함으로 흘러간다.

## 범위: 비코드 산문, 코드 주석 아님

이 hook은 Markdown을 담당한다.
**코드 주석**에 대한 같은 두 관심사는 [`comment-discipline`](comment-discipline) hook에 있다: 그 hook은 편집이 추가하는 모든 주석에 발동하고, 자신의 line-break 검사를 그 핸드오프에 접는다.
두 매체를 두 hook으로 나누므로 코드 편집은 `comment-discipline`만, Markdown 편집은 이것만 발동한다 — 절대 double-fire 하지 않는다.
여기서 line-break 검출이 정적으로 남는 이유는, 문장부호 기반 검사가 Markdown에는 충분히 언어 무관하기 때문이다(한국어 문장도 `다.`/`요.`로 끝난다).

## 무엇을 하나

모든 Markdown 편집에서 추가된 텍스트만 검사하고(Write의 content, Edit/MultiEdit의 새 문자열 — 파일의 기존 산문은 발동시키지 않는다), 다음 둘 중 하나 또는 둘 다를 내보낼 수 있다:

- **line breaks** — 줄이 문장 종결 부호 없이 끝나고 다음 줄이 같은 문장을 이어가면 플래그된다. block은 모든 hard break가 문장 경계에 오도록 줄을 다시 합칠 것을 요구한다. 한 줄에 여러 문장이 있어도 되고, 문장 뒤에서 끊는 것은 허용되되 강제는 아니다. 유일한 예외는 줄 폭에 에러를 내는 프로젝트 린터·포매터다 — 도구 이름을 대고 감김을 유지한다.
- **conciseness** — 추가된 산문이 문장 수 게이트(`CONCISE_SENTENCE_GATE`, 기본 6)를 넘으면, block은 모델에게 그것을 회의적인 편집자로 다시 읽고 이웃을 되풀이하거나 뻔한 말을 하거나 hedging 하는 문장을 쳐내라고 요구한다. 문서는 산문이 있는 게 정상이므로 지침은 다듬기이지, `comment-discipline`이 주석에 적용하는 기본-삭제가 아니다. 간결함은 정적으로 판단할 수 없어서 게이트는 *언제* 물을지만 정하고, 판단은 모델이 한다 — 그래서 어떤 언어에서도 동작한다.

## 무엇을 건너뛰나

- Markdown이 아닌 파일 (코드는 `comment-discipline`으로, 나머지는 무시).
- fenced code block과 YAML frontmatter.
- 구조 줄: 제목, 표 행, fence 마커, `---` 구분선 — break 검사와 문장 수 계산 모두에서.
- 다음 줄이 새 블록(제목, 리스트 항목, 표, 인용, fence)을 시작하는 줄 — 그 끊김은 감김이 아니라 실제 경계다.

## 알려진 한계 (검토 후 수용)

파서가 아니라 휴리스틱이다.
종결 부호 없는 약어로 정당하게 끝나는 문장이나, 의도적인 semantic line break 스타일은 플래그된다 — verdict 방식의 block이라 사유를 대고 유지할 수 있다.
문장 수는 종결 부호로 세므로 `e.g.` 같은 약어는 수를 올리고 부호 없는 스타일은 내린다; 게이트는 정확한 수가 아니라 대략적인 분량 신호다.
Edit 조각은 내부만 검사하므로, 조각의 마지막 줄이 파일의 기존 텍스트와 문장 중간에서 이어지는 경우는 놓친다. 지배적 실패(새 문단 전체를 감거나 쏟아붓는 것)는 커버된다.
