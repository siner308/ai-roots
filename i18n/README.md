# i18n — 한국어 미러

`i18n/ko/`는 `rules/`, `skills/`, `agents/`, 그리고 `hooks/` 문서(`.md`)의 한국어 번역본입니다. 영어 원본을 읽기 어려울 때 내용을 확인하는 용도예요. hook은 문서만 미러하고 스크립트(`.py`/`.json`)는 코드라 미러하지 않습니다.

## 왜 별도 트리인가

`rules/`는 디렉터리 통째로 `~/.claude/rules/ai-roots`에 symlink되고, 그 안의 모든 `.md`가 매 세션 context에 자동으로 올라갑니다. 한국어 파일을 `rules/` 안에 같이 두면 영어+한국어가 둘 다 로드돼서 context가 두 배가 돼요. `agents/`도 `install.sh`가 `*.md`를 잡아 symlink하므로 같은 폴더에 두면 유령 agent가 생깁니다.

그래서 한국어는 symlink 대상 바깥인 `i18n/ko/` 아래에 둡니다. `install.sh`는 `i18n/`를 아예 보지 않으니 한국어가 `~/.claude`로 샐 일이 구조적으로 없습니다.

## 규칙

- **영어가 정본(source of truth)입니다.** Claude는 영어만 읽어요. 동작을 바꾸려면 영어 원본을 고치세요.
- **한국어는 읽기 전용 미러입니다.** 사람이 내용을 볼 때만 씁니다.
- 영어 원본을 추가/수정하면 `i18n/ko/`의 같은 경로도 맞춰주세요.

## 동기화 확인

```sh
./i18n/check-sync.sh
```

영어 파일에 대응하는 한국어 미러가 없거나, 원본이 사라진 한국어 미러가 있으면 알려줍니다.
